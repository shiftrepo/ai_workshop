## 1. 設計コンセプト

### 「セル内複数条件」とは

通常の2軸マトリックスは「縦×横」で1つの値（基本料金）を返すだけですが、  
本設計では **各セルが複数の if/case 分岐ルールを保持** します。

| 条件の種類 | 内容 | if/case イメージ |
|---|---|---|
| **セル固有の基本料金** | 縦軸×横軸の組み合わせで決まる基本料金 | `case 産廃許可×法人: base = ¥220,000` |
| **セル固有の追加料金** | その組み合わせでのみ発生する固定追加料金 | `if 本セル → +¥25,000 加算（事務所実地確認料）` |
| **係数 if/case 分岐** | 優先度・複雑度の係数をセルごとに差し替え | `if 優先度係数 > ×1.50 → case: ×1.50 に差替` |
| **フラグ if/case 分岐** | 環境評価・共同申請・翻訳フラグの適用可否 | `if 環境評価フラグON → case: 不適用（このセルは使用不可）` |
| **割引 if/case 分岐** | リピーター・パッケージ割引の上限をセルで差し替え | `if 割引率 > 10% → case: 10% に差替` |

### 設計構造

```
MATRIX[申請種別A〜F][申請者区分 個人/法人/行政]   ← テーブル駆動
    └─ CellRule（セルごとの if/case 分岐ルール一式）
         ├─ base_fee                    # セル固有基本料金
         ├─ cell_surcharge              # if 本セル → 加算する固定追加料金
         ├─ max_priority_multiplier     # if 優先度係数 > 閾値 → case: 閾値に差替
         ├─ max_complexity_multiplier   # if 複雑度係数 > 閾値 → case: 閾値に差替
         ├─ allow_environmental         # if 環境評価ON → case: 不適用
         ├─ allow_joint                 # if 共同申請ON → case: 不適用
         ├─ allow_translation           # if 翻訳ON    → case: 不適用
         ├─ max_repeat_discount         # if 割引率 > 閾値 → case: 閾値に差替
         └─ max_package_discount        # if 割引率 > 閾値 → case: 閾値に差替
```

## 2. テーブル駆動 と パイプライン の役割分担

本システムは **テーブル駆動** と **パイプライン** を組み合わせた設計です。  
それぞれの役割は明確に分離されています。

```
【テーブル駆動】 ─ 「どのルールを使うか」を決める辞書
    ↓
    MATRIX[申請種別][申請者区分] → CellRule（9つの if/case 分岐ルールを取得）
    REGION_TABLE[地域]          → 地域係数を取得
    PRIORITY_TABLE[優先度]      → 優先度係数を取得
    COMPLEXITY_TABLE[複雑度]    → 複雑度係数を取得
    SCALE_TABLE[事業規模]       → 規模係数を取得
    SEASON_TABLE[提出月]        → 季節係数を取得

【パイプライン】 ─ 「どの順序でどう変換するか」を定義する処理の連鎖
    ↓
    Step 0 → Step 1 → Step 2 → … → Step 11（各Stepでテーブル駆動の値を使用）
```

### 役割の違い

| 観点 | テーブル駆動（本システムでの役割） | パイプライン（本システムでの役割） |
|---|---|---|
| **目的** | 申請種別×申請者区分の組み合わせから if/case ルール一式を取得 | Step順に料金を積み上げ・変換する |
| **変更の単位** | セルのルール（行・列を追加すれば新種別・新区分に対応） | Stepを追加・差替えすれば新しい計算ロジックを挿入 |
| **コードの見え方** | `MATRIX["F: 産廃許可"]["法人"]` → ルール辞書が返る | `run_pipeline(cell_rule, input_params)` → 最終料金が返る |
| **テストの単位** | テーブルの各セル（18セル）が独立してテスト可能 | 各Stepが独立して単体テスト可能 |

## 7. 割引テーブル

> ⚠️ **割引率がセルルールの上限を超える場合は `if > 上限 → case: 上限値に差替`**

### ① リピーター割引テーブル（Step 9 で参照）

| 過去申請件数 | 割引率 | セル if/case 分岐 |
|---|---|---|
| 0件 | 0% | — |
| 1〜2件 | 5% | — |
| 3〜5件 | 10% | `if 補助金×個人 かつ > 10% → case: 10%` |
| 6〜10件 | 15% | `if 個人系全般 かつ > 15% → case: 15%` |
| **11件以上** | **20%** | 法人系: 分岐なし（20%まで） |

### ② パッケージ割引テーブル（Step 10 で参照）

| 同時申請件数 | 割引率 | セル if/case 分岐 |
|---|---|---|
| 1件 | 0% | — |
| 2件 | 8% | — |
| 3〜4件 | 15% | `if 行政系 かつ > 10〜15% → case: 上限値に差替` |
| **5件以上** | **20%** | `if 行政系 かつ > 15% → case: 15%` |

### 割引適用ルール（パイプラインの順序）

1. 割引は「小計（Step 8 確定後）」に対して適用
2. **Step 9: リピーター割引を先に適用** → `if 割引率 > cell.max_repeat_discount → case: cell.max_repeat_discount に差替`
3. **Step 10: パッケージ割引を後から適用**（積み重ね型）→ `if 割引率 > cell.max_package_discount → case: cell.max_package_discount に差替`
4. 割引後は **100円単位で切り上げ** → 消費税10%加算

## 8. 算出パイプライン（テーブル駆動との組み合わせ）

> 🟦 = **テーブル駆動** で値を取得するStep  
> 🟧 = **テーブル駆動 + CellRule の if/case 分岐** が発動するStep  
> ⬜ = 純粋な計算・確定Step

```
入力パラメータ
  （申請種別, 申請者区分, 地域, 優先度, 複雑度, 事業規模, 提出月, フラグ群, 書類枚数, 割引情報）
  ↓
┌─────────────────────────────────────────────────────────────────────┐
│ 🟦 Step 0: CellRule 取得【テーブル駆動】                              │
│   MATRIX[申請種別][申請者区分] → 9つの if/case ルールを取得           │
│   ← ここでセル固有の全分岐ルールが確定する                            │
└─────────────────────────────────────────────────────────────────────┘
  ↓
┌─────────────────────────────────────────────────────────────────────┐
│ 🟦 Step 1: 基本料金 × 地域係数【テーブル駆動】                        │
│   base_fee × REGION_TABLE[地域]                                      │
└─────────────────────────────────────────────────────────────────────┘
  ↓
┌─────────────────────────────────────────────────────────────────────┐
│ 🟧 Step 2: 優先度係数 適用【テーブル駆動 + if/case 分岐】             │
│   × PRIORITY_TABLE[優先度]                                           │
│   if 係数 > cell.max_priority_multiplier                             │
│     → case: cell.max_priority_multiplier に差替                      │
└─────────────────────────────────────────────────────────────────────┘
  ↓
┌─────────────────────────────────────────────────────────────────────┐
│ 🟧 Step 3: 複雑度係数 適用【テーブル駆動 + if/case 分岐】             │
│   × COMPLEXITY_TABLE[複雑度]                                         │
│   if 係数 > cell.max_complexity_multiplier                           │
│     → case: cell.max_complexity_multiplier に差替                    │
└─────────────────────────────────────────────────────────────────────┘
  ↓
┌─────────────────────────────────────────────────────────────────────┐
│ 🟦 Step 4: 事業規模係数 適用【テーブル駆動】                          │
│   × SCALE_TABLE[事業規模]                                            │
│   （法人のみ。if 行政 → case: MICRO 固定。個人は ×1.0）              │
└─────────────────────────────────────────────────────────────────────┘
  ↓
┌─────────────────────────────────────────────────────────────────────┐
│ 🟦 Step 5: 季節係数 適用【テーブル駆動】                              │
│   × SEASON_TABLE[提出月]                                             │
└─────────────────────────────────────────────────────────────────────┘
  ↓
┌─────────────────────────────────────────────────────────────────────┐
│ 🟧 Step 6: セル固有追加料金 加算【if/case 分岐】                      │
│   if 本セル → + cell.cell_surcharge（0円〜+¥35,000）                 │
└─────────────────────────────────────────────────────────────────────┘
  ↓
┌─────────────────────────────────────────────────────────────────────┐
│ 🟧 Step 7: フラグ追加料金 加算【if/case 分岐 × 複数フラグ】           │
│   if allow_environmental=False → case: 環境評価料 適用しない          │
│   if allow_joint=False        → case: 共同申請料 適用しない           │
│   if allow_translation=False  → case: 翻訳料     適用しない           │
│   + 書類超過料金（段階 case 分岐）                                    │
│   + 補正対応料（件数 case 分岐）                                      │
└─────────────────────────────────────────────────────────────────────┘
  ↓
┌─────────────────────────────────────────────────────────────────────┐
│ ⬜ Step 8: 小計確定                                                   │
│   係数後金額 + 追加料金合計 = 小計                                    │
└─────────────────────────────────────────────────────────────────────┘
  ↓
┌─────────────────────────────────────────────────────────────────────┐
│ 🟧 Step 9: リピーター割引 控除【テーブル駆動 + if/case 分岐】         │
│   小計 × (1 - REPEAT_TABLE[過去件数])                                │
│   if 割引率 > cell.max_repeat_discount                               │
│     → case: cell.max_repeat_discount に差替                          │
└─────────────────────────────────────────────────────────────────────┘
  ↓
┌─────────────────────────────────────────────────────────────────────┐
│ 🟧 Step 10: パッケージ割引 控除【テーブル駆動 + if/case 分岐】        │
│   Step9後 × (1 - PKG_TABLE[同時件数])                                │
│   if 割引率 > cell.max_package_discount                              │
│     → case: cell.max_package_discount に差替                         │
└─────────────────────────────────────────────────────────────────────┘
  ↓
┌─────────────────────────────────────────────────────────────────────┐
│ ⬜ Step 11: 最終確定                                                  │
│   100円単位切り上げ → 消費税10% 加算 = 最終料金（税込）              │
└─────────────────────────────────────────────────────────────────────┘
```

## 10. 最大・最小料金レンジ

> 最小条件: 過疎地 × 通常 × 簡易 × MICRO × 8月 × 書類3枚 × フラグ全OFF × リピーター11件以上 × パッケージ5件以上

|---|---|---|---|---|
| 建設工事×個人 | ¥18,200 | ¥571,200 | 約31倍 | 優先度・リピーター割引に分岐あり |
